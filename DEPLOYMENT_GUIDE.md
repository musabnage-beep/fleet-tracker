# دليل التشغيل والتسليم - Fleet Tracker
# ==========================================

## المتطلبات الأساسية

### على جهازك (للبناء):
1. **Node.js** (الإصدار 18 أو أحدث): https://nodejs.org
2. **Git**: https://git-scm.com
3. **حساب Expo**: مجاني من https://expo.dev

### على جهاز الشركة (للتشغيل):
1. **Node.js** فقط (لتشغيل السيرفر)
2. هواتف أندرويد (للموظفين والأدمن)

---

## الخطوة 1: تغيير اسم الشركة والشعار

### تغيير اسم التطبيق والشركة:
1. افتح الملف: `mobile/constants/config.ts`
2. غيّر `appName` لاسم التطبيق الذي تريده
3. غيّر `companyName` لاسم الشركة

### تغيير الشعار:
1. صمم أيقونة 1024x1024 بيكسل بصيغة PNG
2. استبدل الملف: `mobile/assets/icon.png`
3. صمم شاشة البداية (splash) واستبدل: `mobile/assets/splash-icon.png`

### تغيير ألوان التطبيق:
- افتح `mobile/constants/colors.ts` وعدّل الألوان حسب الرغبة

---

## الخطوة 2: إعداد السيرفر (الخادم)

السيرفر هو الكمبيوتر الذي سيخزن البيانات. يمكن أن يكون:
- كمبيوتر مكتبي في الشركة (يجب أن يظل مشغلاً)
- أو VPS رخيص (5$ شهرياً من DigitalOcean أو Hetzner)

### على كمبيوتر السيرفر:

```bash
# 1. ثبّت Node.js من https://nodejs.org

# 2. انسخ مجلد fleet-tracker/server إلى الكمبيوتر

# 3. افتح Terminal/CMD في مجلد server

# 4. ثبّت المكتبات
npm install

# 5. شغّل السيرفر
npm start
```

سيظهر:
```
========================================
  Fleet Tracker Server
  Running on port 3000
  http://localhost:3000
========================================

Default users created:
  Admin: admin / admin123
  Employee: employee / employee123
```

### معرفة عنوان IP السيرفر:
- **Windows**: افتح CMD واكتب `ipconfig` (ابحث عن IPv4 Address)
- **Mac/Linux**: افتح Terminal واكتب `ifconfig` أو `ip addr`
- مثال: `192.168.1.100`

### تشغيل السيرفر بشكل دائم (مهم!):
```bash
# ثبّت pm2 لتشغيل السيرفر في الخلفية
npm install -g pm2

# شغّل السيرفر
pm2 start src/index.js --name fleet-tracker

# اجعله يبدأ تلقائياً عند إعادة تشغيل الكمبيوتر
pm2 startup
pm2 save
```

## الخطوة 3: تحديث عنوان السيرفر في التطبيق

**مهم جداً** - قبل بناء الـ APK:

1. افتح `mobile/constants/config.ts`
2. غيّر `serverHost` لعنوان IP السيرفر:

```typescript
serverHost: '192.168.1.100',  // غيّره لعنوان السيرفر الحقيقي
serverPort: '3000',
```

## الخطوة 4: بناء ملف APK

### الطريقة 1: EAS Build (مجانية - مُوصى بها)

```bash
# 1. ثبّت أدوات Expo
npm install -g eas-cli

# 2. سجّل دخول لحساب Expo
eas login

# 3. انتقل لمجلد التطبيق
cd mobile

# 4. اربط المشروع
eas init

# 5. ابنِ الـ APK (سيتم البناء على سيرفرات Expo مجاناً)
eas build -p android --profile preview
```

بعد الانتهاء (15-30 دقيقة)، سيظهر رابط لتحميل ملف APK.

### الطريقة 2: البناء المحلي (تحتاج Android Studio)

```bash
# 1. ثبّت Android Studio وأعده
# 2. من مجلد mobile:
npx expo prebuild -p android
cd android
./gradlew assembleRelease
```

الملف سيكون في: `android/app/build/outputs/apk/release/app-release.apk`

## الخطوة 5: التسليم للشركة

### ما تسلمه:
1. **ملف APK** - التطبيق للهواتف
2. **مجلد server** - السيرفر للكمبيوتر

### خطوات التثبيت عند الشركة:

#### تثبيت السيرفر:
1. انسخ مجلد `server` لكمبيوتر الشركة
2. ثبّت Node.js
3. افتح CMD في مجلد server واكتب:
   ```
   npm install
   npm install -g pm2
   pm2 start src/index.js --name fleet-tracker
   pm2 startup
   pm2 save
   ```

#### تثبيت التطبيق على الهواتف:
1. أرسل ملف APK عبر واتساب أو بلوتوث أو فلاش ميموري
2. على كل هاتف: افتح الملف واضغط "تثبيت"
3. قد تظهر رسالة "تطبيق غير معروف" - اضغط "السماح" أو "Install Anyway"

## الخطوة 6: الاستخدام

### بيانات الدخول الافتراضية:
- **المدير**: admin / admin123
- **الموظف**: employee / employee123

### طريقة العمل:

#### المدير يقوم بـ:
1. تسجيل الدخول كمدير
2. إضافة السيارات (أرقام اللوحات) من تبويب "السيارات"
3. إنشاء وردية يومية من تبويب "الورديات" (يحدد التاريخ ويختار السيارات)
4. مراجعة التقارير من تبويب "التقارير"
5. تحميل التقرير كـ PDF

#### الموظف يقوم بـ:
1. تسجيل الدخول كموظف
2. يرى وردية اليوم والسيارات المطلوبة
3. يضغط "بدء المسح"
4. يفتح الكاميرا (مع تشغيل الفلاش للإضاءة المنخفضة)
5. يدخل أرقام اللوحات يدوياً أو بالمسح
6. يضغط "إنهاء المسح" لإرسال التقرير

## ملاحظات مهمة

### الأمان:
- غيّر كلمات المرور الافتراضية فوراً من شاشة "الإعدادات"
- السيرفر يعمل على الشبكة المحلية فقط (آمن)

### النسخ الاحتياطي:
- قاعدة البيانات في: `server/data/fleet.db`
- انسخ هذا الملف بانتظام للحفاظ على البيانات

### استكشاف الأخطاء:
- **التطبيق لا يتصل**: تأكد أن عنوان IP صحيح وأن السيرفر شغال
- **الهاتف والسيرفر على نفس الشبكة**: يجب أن يكونا على نفس WiFi
- **السيرفر توقف**: شغله مجدداً بـ `pm2 restart fleet-tracker`

### المسح الضوئي:
- شغّل الفلاش في الأماكن المظلمة (الزر أعلى يسار الكاميرا)
- استخدم "الإدخال اليدوي" إذا الكاميرا لم تقرأ اللوحة
- قرّب الكاميرا من اللوحة (30-50 سم)
